💡
Key insight: Stable Diffusion runs denoising in latent space (compressed), while older models like DeepFloyd IF work in pixel space (uncompressed). Latent space makes SD faster and more VRAM-efficient — that's why it runs on consumer hardware and Google Colab's free T4 GPU.

Fine-tuning & Adaptation Methods
Ways to teach a model new concepts, styles, or subjects without training from scratch.
Method What it does Training Data VRAM needed Output size Colab Free? Paper
LoRA
Low-Rank Adaptation		 Adds small trainable matrices alongside frozen model weights. Learns style/character delta. 15–100 images 8–12GB ~50MB–500MB Yes ✅ arxiv:2106.09685 ↗
DreamBooth Fine-tunes the entire model on a specific subject using a rare token. Very strong subject fidelity. 3–30 images 16–24GB Full model (~4GB+) Tight arxiv:2208.12242 ↗
Textual Inversion Learns a new word embedding (token) that represents a style or object. Doesn't change model weights. 3–10 images 6–8GB ~5–10KB Yes ✅ arxiv:2208.01618 ↗
ControlNet Adds spatial conditioning (pose, edge, depth, segmentation) to control image layout precisely. Large dataset (pre-trained) 8–12GB ~1GB per module Yes (inference) arxiv:2302.05543 ↗
HyperNetworks A small secondary network that generates weight modifications for the main model in real-time. 100–1000 images 12GB+ ~100MB Possibly arxiv:1609.09106 ↗
Full Fine-tuning Retrains all model weights. Maximum capability but expensive and needs huge data. 10K–millions of images 40GB+ (A100) Full model (~4–12GB) No ❌ —
Adapters Adds small bottleneck layers between frozen transformer blocks. Like LoRA but separate modules. 100–10K images 8–16GB ~10–100MB Possibly arxiv:1902.00751 ↗
IP-Adapter Image-based conditioning — use a reference image to guide the style of generation. Pre-trained (use off-shelf) 8–10GB ~150MB Yes ✅ arxiv:2308.06721 ↗
Prefix Tuning Prepends trainable "virtual tokens" to the transformer input. NLP-origin but used in vision-language. Medium Low Very small Yes arxiv:2101.00190 ↗
P-Tuning Uses a small trainable MLP to generate prompt embeddings rather than discrete tokens. Small–Medium Low Very small Yes arxiv:2103.10385 ↗
🎓
For anime style on Google Colab (Free T4): LoRA is your best friend. Train with 15–30 Danbooru images using kohya_ss or Diffusers trainer. Keep batch size=1, gradient accumulation=4, rank=16 or 32. Training takes ~30–60 min on T4. Use Textual Inversion if you want an even smaller output file.

Section E — Must Know
Core Concepts Explained
The building blocks everyone assumes you know. Don't skip these — they unlock everything else.
🌊

Diffusion Process

Training: take a real image, add Gaussian noise over T steps until it's pure static. Inference: start from pure noise, remove noise step-by-step guided by your prompt until an image emerges.

📦

Latent Space

A compressed mathematical representation of images. VAE compresses 512×512 pixels → 64×64 latent tensors (8× smaller). All denoising happens here — making SD much faster than pixel-space models.

🔗

Cross-Attention

The mechanism that links your text embedding to the image being generated. Each UNet block asks "which parts of my text description are relevant to these image regions?" LoRA primarily modifies these layers.

🔤

CLIP

Contrastive Language-Image Pretraining by OpenAI. Learns to align text and image in shared embedding space. SD 1.5 uses ViT-L/14 CLIP. SDXL uses both OpenCLIP-ViT/G AND CLIP-ViT/L for better understanding.

🕸️

UNet

U-shaped neural network with encoder-decoder architecture + skip connections. The main workhorse of SD — repeatedly predicts and removes noise from the latent image. Contains the cross-attention layers LoRA modifies.

🗜️

VAE

Variational Autoencoder. Encodes real images into latent space for training/img2img, decodes latents back to pixels at the end. A better VAE = crisper final images. The SDXL VAE is improved over SD 1.5.

🎲

Sampler / Scheduler

Controls the math of how noise is removed at each step. Popular: DDIM (fast, deterministic), Euler (simple), DPM++ 2M Karras (quality+speed balance), DDPM (original). Fewer steps = faster but lower quality.

🌱

Seed

The random number that initializes the noise pattern. Same seed + same prompt + same settings = identical image every time. Use a fixed seed to iterate on composition while changing other settings.

📏

CFG Scale (Guidance)

Classifier-Free Guidance. Ranges 1–20+. Low (1–5): creative, may ignore prompt. Medium (7–9): balanced. High (12+): over-saturated, rigid. Start at 7 and adjust. SDXL works better at lower CFG than SD 1.5.

🚫

Negative Prompt

Words that push the generation away from unwanted content. Classic starter: "blurry, deformed, ugly, bad anatomy, extra limbs, watermark, signature, text, cropped". Very effective in SD 1.5 and SDXL.

🔀

LoRA Rank

Rank (r) controls LoRA expressiveness. Rank=4 is tiny. Rank=128 is large. Higher rank = more learning capacity but larger file + possible overfitting. Rank 16–32 is sweet spot for most styles.

📐

Attention Slicing

Memory optimization: process attention in chunks instead of all at once. Reduces VRAM usage by ~30% at the cost of slightly slower generation. Essential for running SDXL on Colab's free T4 GPU.

💾
Essential

Mount Google Drive first

Save models and outputs to Drive so you don't lose them when the session resets. Use drive.mount('/content/drive') at the start of every notebook.

⚡
Must do

Always enable GPU Runtime

Runtime → Change runtime type → Hardware accelerator → GPU (T4). Without this, SD takes hours instead of seconds. Don't forget to do this every session.

🔧
VRAM tip

Use fp16 and attention slicing

Always load models with torch_dtype=torch.float16. Call pipe.enable_attention_slicing(). This halves VRAM and lets SDXL run on the T4.

🕐
Time limit

Sessions reset every ~1–3 hours

Free Colab disconnects after ~1 hour idle or 3–12 hours active. Download your outputs to Drive frequently. Don't rely on Colab disk for long LoRA training runs.

📥
HuggingFace

Download models from HuggingFace

Use from diffusers import StableDiffusionPipeline and pass the HF model ID. Models cache in /root/.cache — link this to Drive to avoid re-downloading each session.

🎯
Start here

Best starter combo

SD 1.5 + Diffusers library on free T4. Then try SDXL with fp16. For LoRA training: use kohya_ss notebook with SD 1.5. For UI: ComfyUI + Pinggy tunnel.

⚠️
Warning

Don't abuse the free tier

Google may throttle GPU access if you use Colab heavily over multiple days. They prioritize users who haven't used GPU recently. If denied GPU, wait a day or use CPU for setup steps.

🧪
For training

LoRA training tips for T4

Resolution: 512 (SD1.5) or 768 (SDXL). Batch size: 1. Grad accumulation: 4–8. Steps: 1000–2000. Use 8-bit Adam optimizer (use_8bit_adam=True). Enable gradient checkpointing.
